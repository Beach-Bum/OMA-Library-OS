# Deploying oma.ooo

The public site **https://oma.ooo** is served by the Vercel project **`site`**.

- Vercel project: `site` (`prj_v8yG09cH0WIyZZcYXeDBG3YYRdoF`)
- Vercel team: `jimjoes-projects-b0a981bc` (`team_n9UkMo8atZuKVQ3o2YoCnpjU`)
- Production domain: `oma.ooo`
- **Deploys are CLI-only.** There is no GitHub → Vercel auto-deploy. Merging to
  `main` does **not** update the site. Someone must run `vercel deploy`.
- Deploy root is the **`site/`** directory (it serves `/` and `/docs/*`).
- The machine that normally deploys (and has the Vercel auth) is the Alienware,
  under `~/Projects/phi-mu-lambda`.

## Procedure

```bash
cd ~/Projects/phi-mu-lambda
git fetch origin
git checkout main
git reset --hard origin/main          # safe: no local-only commits are kept

python3 build.py                       # regenerate site/ from _content-backup/*.mdx

# 1. site/ must be reproducible from source (no drift)
git diff --quiet site/ \
  && echo "site/ matches committed — OK" \
  || { echo "ABORT: site/ drifted from build.py output"; exit 1; }

# 2. corrected register labels present, old form absent
grep -q 'Λ — Lambda — Function' site/docs/registers.html \
  && ! grep -q 'Λ (Lambda)' site/docs/registers.html \
  && echo "labels OK" \
  || { echo "ABORT: label check failed"; exit 1; }

# 3. preview deploy — open the printed *.vercel.app URL and check
#    /docs/registers.html shows "Λ — Lambda — Function"
cd site && vercel deploy

# 4. only after the preview looks right, promote to production (oma.ooo)
vercel deploy --prod
```

## Verify after deploy

Open <https://oma.ooo/docs/registers.html>. It must show
`Λ — Lambda — Function` (not `Λ (Lambda)`), and
<https://oma.ooo/docs/reading.html> must still contain its
`og:` / `twitter:` SEO meta tags.

## Notes

- `build.py` generates the full SEO `<head>` from each page's `seo:` frontmatter
  in `_content-backup/*.mdx`. Page body content also comes from those `.mdx`
  files. Edit the `.mdx`, re-run `build.py`, commit `site/` + `_content-backup/`.
- `site/index.html`, `robots.txt`, `sitemap.xml`, `style.css` are **not**
  generated by `build.py` — they are hand-maintained static assets.
